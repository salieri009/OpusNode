## 1. Glossary

### Core Platform Terms

| Term | Definition |
|:-----|:-----------|
| **Assessment** | AI-driven conversational interview to evaluate user's technical skills across multiple domains (backend, frontend, AI/ML, etc.) |
| **Matching** | Algorithmic process to form optimal learning groups based on skill complementarity, shared goals, and availability |
| **Curriculum** | Personalized learning roadmap generated by AI, tailored to user's current skill level and learning objectives |
| **Workspace** | Collaborative environment for group projects, featuring Kanban boards, file sharing, and real-time chat |
| **Portfolio** | Auto-generated professional showcase of completed projects, including tech stack, contributions, and screenshots |
| **Active Learning Group (ALG)** | Group with â‰¥3 meaningful interactions (chat, commits, tasks) within a rolling 7-day period |

### Technical Terms

| Term | Definition |
|:-----|:-----------|
| **Vector DB** | Database optimized for storing and querying high-dimensional embeddings (e.g., Milvus, Pinecone) |
| **LLM** | Large Language Model - AI system trained on vast text corpora (e.g., GPT-4, Claude 3.5) |
| **Embedding** | Numerical vector representation of text/code used for semantic similarity calculations |
| **Cosine Similarity** | Metric measuring similarity between two vectors, used in matching algorithm |
| **JSONB** | PostgreSQL column type for storing JSON data with indexing support |
| **Prerequisite** | Dependency relationship between learning topics (e.g., JavaScript â†’ React) |
